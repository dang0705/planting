# 04 视觉证据接入、正式证据与诊断方向

> 文档口径：本组文档基于 `Archive 2.zip` 中可见的 CloudBase 云函数代码、`rules.zip` 中的实施规范、以及现有粗文档 `diagnosis-runtime-code-logic.md` 整理。中文概念优先，英文函数名、字段名、路径只用于定位代码。
>
> 重要限制：压缩包内未发现 uni-app/小程序前端页面源码，因此前端实现只能按后端接口契约、返回结构和小程序接入规范反推，不写成“已确认源码行为”。

## 一、视觉接入位置

视觉接入分三层：`app.js` 的 `extractVisualSymptoms()` / `extractVisualSymptomsSafely()` 负责入口调用；`services/visual-diagnosis-service.js` 负责视觉批次分析、聚合和持久化；`domain/diagnosis-engine.js` 把视觉聚合结果转成运行时可使用的正式证据、视觉候选和诊断方向。

视觉结果不是直接诊断结果。它必须经过运行时归一、证据准入、方向归纳和输出守卫。

## 二、视觉输入结构

开始诊断支持：

- `images` / `imageInputs`：推荐结构化输入，每张图可带部位、顺序、用户声明器官、压缩信息。
- `imageIds`：旧式图片 ID 列表。
- `image`：单图兼容字段。

归一后字段包括 `imageRef`、`inputSlotType`、`orderIndex`、`inputSlotOrder`、`inputSlotLabel`、`userDeclaredOrganType`、`userDeclaredOrganConfidence`、`uploadCompression`。

## 三、视觉聚合结果的三种去向

### 1. 正式观察证据

通过 `buildObservedEvidenceSetFromVisualAggregateResult()` 进入 `observedEvidenceSet`。只有这类证据才可以参与候选 outcome 与输出资格判断。正式证据关注 `symptomKey`、`sourceType`、`enteredRuntime`、`currentStatus`、`originVisualCallBatchId`、`firstSeenStage`。

### 2. 视觉候选症状

通过 `collectVisualCandidateSymptoms()` 形成。视觉候选用于提出确认题，不能等同正式证据。

### 3. 视觉路由提示

通过 `resolveVisualRouteContext()` 形成。视觉路由提示影响首轮动作，例如 `ask_first`、`retake_first`、`uncertain_prepare`。路由提示只影响流程，不是结论。

## 四、正式证据集合的意义

正式证据集合是系统的“证据账本”，用于回答：哪些事实已经被允许进入诊断、这些事实来自视觉/问诊/派生/旧链路、这些事实当前是否还有效。后续任何具体问题输出，都应能回溯到正式证据集合。

## 五、派生证据

`utils/derived-evidence.js` 负责生成派生证据。派生证据用于补充方向判断，但不能替代正式视觉/问诊事实，不能单独支撑具体问题输出，必须保留来源和推导路径。

## 六、诊断方向归纳

`utils/diagnosis-directions.js` 负责构建诊断方向。方向不是问题本身，而是问题族或诊断路径。它用于限定追问范围、限定输出问题范围、防止候选 outcome 从无关方向越界，并为审计提供“为什么沿这个方向问”的基础。

## 七、池外视觉异常

池外视觉异常指视觉发现了当前症状池无法稳定承接的异常。当前代码区分完全池外且无映射、弱池外提示、已有审计 proxy mapping。池外机制的目的不是扩大误判，而是保护系统边界：看不懂就承认看不懂，等审计后再纳入规则。

## 八、高风险视觉证据类型

| 类型 | 风险 | 正确处理 |
|---|---|---|
| 黄叶 | 病因很多，缺水、积水、缺素、光照、自然老化都可能 | 必须走黄叶分流 |
| 孔洞/缺损 | 可能是虫咬、病斑脱落、机械伤、旧伤 | 必须区分虫害痕迹、病斑、机械损伤 |
| 不规则斑块 | 可能是叶斑、晒伤、药害、机械损伤、水渍等 | 必须追问边界、质地、扩散、颜色 |
| 生长异常 | 可能是虫害、药害、光照、营养、环境压力 | 宽泛异常不能零问诊闭合 |
| 土表小飞虫 | 可进入真菌蚊方向 | 不能由泛化黄点、黏液、虫害信号自动带入真菌蚊 |

## 九、视觉证据维护要求

新增视觉标签时必须回答：是否能进入正式观察证据、是否只是视觉候选、对应哪个 `symptomKey`、特异性等级、能否触发高特异快速收敛、是否需要问诊确认、是否会打开某个诊断方向、是否可能属于池外异常、在不确定结果中如何解释。
