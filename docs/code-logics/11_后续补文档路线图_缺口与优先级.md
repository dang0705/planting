## 二、当前最大缺口

## 四、下一步建议

最应该先补的是 `行动建议体系`。原因是当前系统已经有问题簇、诊断方向、不确定、非问题和具体问题多种结果；行动建议如果只按具体问题绑定，会导致大量合法不确定结果缺少可执行建议。

行动建议体系应明确：具体问题输出时使用问题级建议；问题簇明确但具体问题不稳时使用问题簇建议；不确定但有方向时使用方向级保守建议；不确定且无方向时使用补图/观察建议；非问题时使用观察与维持建议；池外异常时使用保守观察和人工复核建议。
